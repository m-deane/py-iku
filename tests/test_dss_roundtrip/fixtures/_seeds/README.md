# Seed fixtures (hand-authored canonical shapes)

These fixtures are NOT auto-generated by `extract_fixtures.py`. They are
hand-authored canonical-shape fixtures that complement the (very small) set
of inline JSON examples present in the public-doc snapshot.

Each seed JSON has the same envelope as auto-extracted fixtures:

```jsonc
{
  "_meta": {
    "kind": "recipe" | "processor",
    "source_md": "docs/dataiku-reference/.../page.md",
    "source_url": "https://doc.dataiku.com/dss/...",
    "category": "recipes" | "processors",
    "seed": true,
    "doc_lines": "L<start>-L<end>",   // line range backing the canonical shape
    "extracted_by": "manual"
  },
  "payload": { ... }                  // the canonical wire JSON
}
```

The harness loads seeds the same way it loads auto-extracted fixtures. Seeds
are the long-term backstop because most public DSS doc pages enumerate
options as English prose (Markdown bullet lists), not as wire JSON.

When DSS publishes a doc page that DOES contain a JSON example, the
extractor will pick it up automatically and you can retire the
corresponding seed.

## Handover Sessions

### Session 1: Project Overview (1-2 hours)

**Objective**: New owner understands what the project does and why

| Topic | Covered | New Owner Confirms Understanding |
|-------|---------|----------------------------------|
| Business purpose and value | ☐ | ☐ |
| Key stakeholders and consumers | ☐ | ☐ |
| Data sources and their owners | ☐ | ☐ |
| Output deliverables and SLAs | ☐ | ☐ |
| Project history and context | ☐ | ☐ |

**New Owner can explain**:
- [ ] What the project does in one sentence
- [ ] Who uses the outputs and why
- [ ] When the project runs and why that timing

---

### Session 2: Technical Walkthrough (2-3 hours)

**Objective**: New owner understands how the project works

| Topic | Covered | New Owner Confirms Understanding |
|-------|---------|----------------------------------|
| Flow structure and zones | ☐ | ☐ |
| Key recipes and their purposes | ☐ | ☐ |
| Business rules and where implemented | ☐ | ☐ |
| Data lineage for key fields | ☐ | ☐ |
| Scenarios and scheduling | ☐ | ☐ |

**New Owner can**:
- [ ] Navigate to any recipe within 30 seconds
- [ ] Explain what each zone does
- [ ] Trace an output field back to source
- [ ] Identify where a business rule is implemented
